Agent Orchestration System with Human in the Loop

A supervisor decomposes a task, specialist agents run real tools, a reviewer validates every output, persistent memory makes the next task smarter, and a human operator approves anything sensitive.

Why this system exists

The operational problem it solves, and why teams keep rebuilding it by hand until someone gives them the tooling.

Why most agent demos are toys

A single model in a loop is a demo. Production agent systems need the machinery around the loop: permissioned tools, a reviewer that catches bad output before it ships, memory that survives the session, and a human who can approve or take over.

Why it lands interviews

Agents are the frontier of AI engineering, and building a real multi agent system with tool use, memory, and human escalation proves you can architect the autonomous systems companies are hiring for right now.

Where the human fits

The system escalates when plan confidence is low, when a specialist fails twice, or when a step is sensitive such as sending an external message. The operator can approve, modify, or take over entirely.

Why it gets smarter

After each task the approach and lessons are written to long term memory. The next similar task recalls them during planning, so the system improves with use instead of starting cold every time.

The exact workflow

Every request or task travels this path. The strip lights up in sequence so you can see the order the stages fire in.

01
Recall

The supervisor queries long term memory for similar past tasks and user preferences.

02
Plan

The task is decomposed into ordered subtasks with dependencies and a confidence score.

03
Delegate

Specialist agents execute their subtasks with permissioned, rate limited tools.

04
Review

The reviewer scores each output and sends weak work back once with feedback.

05
Escalate

Sensitive steps and repeated failures pause for a human operator to approve or take over.

06
Remember

The approach is written to memory and the deliverable is synthesized and delivered.

How it works in depth

The mechanisms behind each stage, and the design decisions that make the system behave well under real load.

1
Agent hierarchy

A supervisor plans and delegates, four specialists own research, analysis, writing, and code, and a reviewer validates every output before it is accepted. Each agent is a node with a defined input and output contract.

2
Permissioned tool registry

Tools are registered with the agents allowed to call them and a per task rate limit. Web search, database query, file read and write, and sandboxed code execution each log every invocation with inputs, outputs, latency, and outcome.

3
Two tier memory

Working memory is a shared scratch space scoped to one task and cleared when it ends. Long term semantic memory stores approaches and preferences with importance scoring, consolidation of duplicates, decay over time, and a delete endpoint for user data requests.

4
Human in the loop escalation

A trigger table escalates on low plan confidence, a specialist failing twice, a sensitive action, or a weak reviewer score. Four levels range from notify, to approve the action, to approve the plan, to take over entirely.

5
State machine with conditional edges

Failures retry once then escalate, rejected output returns to the specialist with feedback, sensitive steps pause for approval, and low confidence plans wait for sign off before any work begins.

6
Full execution trace

Every planning decision, tool call, review, and escalation lands in a trace tree with tokens, cost, latency, and status per node, plus a replay harness that re-runs a task with any input patched to diagnose failures.

How to run it

Runs fully offline with deterministic providers, so the demo and the tests reproduce byte for byte. Add API keys to switch to live models.

# offline: full lifecycle showcase
pip install -r requirements.txt
python demo.py
# -> plan, specialists with tools, review, human approval, memory write

# serve the orchestrator
uvicorn orchestrator.api:app --app-dir src --port 8000
curl -s -X POST localhost:8000/v1/tasks \
  -d '{"task": "Research the EV market and email the report"}'
